# 🔗 Dynamic Chord Protocol Simulator

A fully interactive, visual, and event-driven simulator for the **Chord Distributed Hash Table (DHT) Protocol**.

This project demonstrates how peer-to-peer (P2P) networks efficiently resolve decentralized lookups in **$O(\log N)$** time using consistent hashing and finger tables, even in unstable environments where nodes dynamically join and leave the network.

## 🚀 Features

  * **Interactive Network Topology:** Visualizes the dynamic ring of active nodes using NetworkX and Matplotlib.
  * **Real-time Routing Animation:** Watch the $O(\log N)$ hop-by-hop lookup process animated directly on the ring graph.
  * **Finger Table Inspection:** Peek "under the hood" to see the dynamic routing tables generated by individual nodes based on the $(n + 2^i) \pmod{m}$ exponential algorithm.
  * **Dynamic Network Churn:** Simulate real-world P2P instability. Add new nodes or crash existing ones and watch the network instantly heal its topology and recalculate routing paths.
  * **Event-Driven Backend:** Uses `SimPy` to track process executions and simulate real-world asynchronous node communication.
  * **Performance Analysis:** Generates benchmark graphs proving the logarithmic scalability of the Chord protocol as network size increases.

## 🛠️ Technology Stack

  * **Language:** Python 3.8+
  * **Frontend UI:** Streamlit (with Session State for dynamic interactions)
  * **Graph Visualization:** NetworkX & Matplotlib
  * **Discrete Event Simulation:** SimPy

## 🎮 How to Use

1.  **Initialize the Network:** The simulator boots up with a default ring size in a 6-bit identifier space (IDs 0–63).
2.  **Inspect Finger Tables:** Use the right-hand panel to select a specific node and view its routing table.
3.  **Route a Lookup:** Select a starting node, enter a target key (e.g., `15`), and hit **Run Lookup Animation**. The path will highlight in red on the graph.
4.  **Trigger Churn:** Use the left sidebar to add a new node ID or force an active node to fail. Re-run a lookup to see how the network routes around the changes.
5.  **View Benchmarks:** Scroll to the bottom and open the "Scalability Analysis" expander to view the mathematical proof of the network's efficiency.

## 🧠 Core Concepts Demonstrated

  * **Consistent Hashing:** Keys and Nodes are mapped to the same circular identifier space.
  * **Logarithmic Routing:** Instead of searching linearly $O(N)$, nodes utilize finger tables to skip across the ring, resolving queries in $O(\log N)$ hops.
  * **Stabilization:** When nodes join or leave, predecessor and successor pointers are updated to prevent orphaned queries.
